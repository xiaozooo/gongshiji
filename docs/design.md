# 产品设计文档

> 基于 2026-03-11 仓库代码整理，描述当前已交付产品形态，而不是早期规划稿。

## 1. 产品定位

- 产品名称：工时记
- 形态：移动优先的本地工时记录 PWA
- 用户：小时工、兼职人员、临时工，以及需要本地快速记工和对账的人
- 核心价值：低操作成本、本地保存、快速汇总、可导入导出

## 2. 信息架构

应用只有三个主页面：

- `#home`：录入工时
- `#history`：看统计与历史记录
- `#settings`：管理岗位和数据

底部导航固定为“记工时 / 历史 / 设置”。

## 3. 当前已实现功能

### 3.1 首页录入

- 选择岗位，默认优先展示并选中默认岗位
- 选择日期，用于当天记录或补录
- 输入开始时间和结束时间
- 录入休息分钟数
- 可选填写临时工价，覆盖岗位默认时薪
- 实时显示净工作分钟数、小时数和预计收入
- 支持“复制上一次记录”作为快速录入入口

### 3.2 录入约束与交互

- 没有岗位或工时无效时，保存按钮禁用
- 结束时间必须晚于开始时间
- 休息时长不能大于等于总时长
- 同一天同岗位已有记录时，弹窗提供“追加 / 合并 / 取消”
- 发生时间重叠时，弹窗要求用户确认是否继续保存

### 3.3 岗位管理

- 自动创建一个“小时工”岗位，时薪初始为 `10`
- 新建岗位：名称、时薪、备注
- 编辑岗位
- 归档岗位
- 恢复岗位
- 设置默认岗位
- 归档岗位与活跃岗位分区展示

### 3.4 历史统计

- 按周查看：周范围 KPI + 趋势 + 每日明细
- 按月查看：月范围 KPI + 趋势 + 每日明细
- 按工作查看：岗位维度工时、收入、记录数、出勤天数
- 支持点击趋势图中的日期做筛选
- 支持长按历史记录进入“编辑 / 删除”操作
- 删除后提供 5 秒撤销条

### 3.5 数据管理

- 设置页导出全部记录为 CSV
- 设置页导入 CSV 记录
- 导入前展示新增数与冲突数
- 导入策略支持“追加”或“覆盖冲突记录”
- 清空全部本地岗位与记录数据

### 3.6 PWA 与离线

- 已集成 manifest 和 Service Worker
- 支持安装到桌面或主屏幕
- 具备离线缓存能力
- 页面顶部有在线/离线状态条

## 4. 当前未交付但代码里有痕迹的能力

这些内容当前未上线：

- 跨天工时拆分：首页录入流程仍禁止 `结束时间 <= 开始时间`
- JSON 备份/恢复：当前设置页未提供入口

## 5. 关键体验规则

- 收入计算优先级：自定义工价 > 岗位默认时薪
- 历史统计全部基于记录内冗余保存的 `jobName` 和 `wage`
- 删除记录默认是软删除，可在短时间内撤销
- 数据全部保存在当前浏览器本地，不自动上传

## 6. 已知限制

- 当前不能直接记录跨午夜班次
- 在线状态从离线切回在线时，状态条文本会消失
- CSV 解析更适合本项目导出的标准格式，对复杂 CSV 的兼容性有限
- 清空数据后不会自动重建默认岗位，需重新进入流程后由启动逻辑补齐
