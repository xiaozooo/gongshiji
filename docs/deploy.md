## 3. 关键实现说明

### 3.1 首页录入

当前录入模型为：

- 日期
- 开始时间
- 结束时间
- 休息分钟
- 自定义工价

净工时计算方式：

```text
净工时分钟 = max(0, 总分钟 - 休息分钟)
净工时小时 = 净工时分钟 / 60
```

### 3.2 历史页

当前历史页已不再是“月历 + 列表”单一模式，而是三种统计模式：

- 周统计
- 月统计
- 工作统计

每种模式都围绕同一批记录做聚合。

### 3.3 设置页

设置页中的岗位管理使用 `renderJobsPanel()` 挂载在同一页面里，并未拆分到独立路由。CSV 导入会先做冲突统计，再由用户决定“追加”还是“覆盖”。

## 4. 已知问题与限制

### 4.1 功能限制

- 结束时间必须晚于开始时间，跨夜班次无法直接记录
- 清空数据后当前页只会刷新岗位面板，不会立即自动重新创建默认岗位
- 在线状态条恢复在线时没有文本提示

### 4.2 工程限制

- `package.json` 没有 `test` 命令
- 仓库内存在未接线路径和未接入工具模块
- CSV 导入仅适配相对标准的表格格式
